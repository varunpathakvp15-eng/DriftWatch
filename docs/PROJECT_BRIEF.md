# DRIFTWATCH — PROJECT BRIEF

## What This Project Is
A policy stress-testing platform where governments simulate the
consequences of policy decisions on 100,000 autonomous AI citizen
agents before those decisions affect real people.

NOT a prediction engine. NOT an oracle.
A crash-test dummy for policy — powered by autonomous agents.

## One-Sentence Pitch
"Driftwatch is an autonomous multi-agent simulation platform
where 100,000 AI citizens with individual memory, reasoning, and social
networks stress-test government policy decisions — before real people
pay the consequences."

## Primary Theme: Agentic & Autonomous Systems

This is the core identity of the submission. Every component is an
argument for what autonomous agent systems can do at scale.

★ The citizen agents are autonomous — they are not responding to
  prompts. They wake every simulation step, retrieve their memory,
  query their social network, reason about their situation, make
  a decision, and broadcast that decision to their connections.
  Nobody instructs them. They act.

★ The Government Agent is autonomous — it monitors the simulation,
  detects emergent resistance before it becomes a crisis, generates
  alternative policy formulations unprompted, and issues early
  warning alerts without human intervention.

★ The Red-Team Agent is autonomous — it continuously probes the
  examination security system looking for vulnerabilities, operates
  independently of the main simulation loop, and files adversarial
  reports that other agents respond to.

★ The Causal Chain Agent is autonomous — it observes simulation
  outcomes, traces decision pathways backward through the agent
  network, and generates human-readable explanations of why outcomes
  occurred without being asked for any specific chain.

★ The entire system produces emergent behaviour — protests, coalitions,
  boycotts, policy reversals — that no individual agent was programmed
  to produce. This is autonomous multi-agent coordination at scale.
  This is the submission.

## Hardware Component (Minimal, Optional)
If the hackathon awards points for hardware integration:

### Option A — Raspberry Pi Policy Terminal (Simplest)
A Raspberry Pi 4 running a stripped-down Driftwatch client.
Physical keyboard input for policy statements. Small display showing
live simulation metrics streaming from the cloud backend.
Makes the "deployable by a district collector" narrative tangible.
Hardware cost: ~₹4,000. Setup time: 4 hours.
What it demonstrates: the platform is not just a web app — it is
a deployable tool that can run in a government office with no
internet-connected laptop required.

### Option B — Sensor-Fed Agent Memory (More Impressive)
A Raspberry Pi with:
- GPS module: feeds real-time location context to nearby agent clusters
- DHT22 temperature/humidity sensor: feeds weather context to
  crisis injection scenarios
- Small OLED display: shows live agent count + current simulation day
The sensors provide live environmental data that seeds agent memory
with current real-world conditions rather than historical estimates.
Hardware cost: ~₹6,500. Setup time: 8 hours.
What it demonstrates: autonomous agents that respond to real-world
sensor inputs — a direct demonstration of the agentic theme.

### Option C — Agent Network Visualiser (Most Visual Impact)
A Raspberry Pi driving a physical LED matrix (8×8 or 16×16).
Each LED represents an agent cluster. Colour = current sentiment
(green → yellow → red). Brightness = decision activity level.
As the simulation runs, the LED matrix visually shows sentiment
cascading through the population in real time.
Hardware cost: ~₹5,500. Setup time: 6 hours.
What it demonstrates: emergent behaviour made physically visible.
Judges see the cascade happen in front of them, not on a screen.

RECOMMENDATION: Option B or C. Option B is more technically credible.
Option C has more immediate visual impact for a live presentation.
If time is limited, Option A is the safe choice.

## Core Problem
India's biggest governance disasters share one thread —
the policy was tested on real people first.

★ NEET 2024: 2.4 million students affected. Examination security
  never stress-tested against adversarial scenarios.
★ Delhi odd-even scheme: reduced traffic by ~0%.
  ₹1,200 crore in economic disruption. Simulated nowhere.
★ 2016 demonetisation: 1.5 million jobs lost in 50 days.
  Informal market ripple effects never modelled.
★ Railway fare revisions: estimated using spreadsheets,
  not simulation.

The current standard is expert opinion and historical averages.
Autonomous agent simulation is systematically better.

## What Makes It Different From Existing Tools
- NetLogo (1999): rule-based agents, lookup table with animation
- AnyLogic: statistical distributions, no cognition or memory
- Mesa: open source ABM, same rule-based limitation
- MIT CityScope: GIS-based geography without psychology

DIFFERENTIATOR (state this exactly):
"The first platform to combine LLM-powered autonomous agents with
persistent memory, social network dynamics, and causal chain
explainability at population scale for government policy
stress-testing."

Never say "world's first." Use the sentence above exactly.

## Why Agentic & Autonomous Systems Is the Right Theme

Most submissions on this theme will build:
- AI assistants that take instructions
- Chatbots with tool use
- Workflow automation pipelines
- Single-agent task runners

Driftwatch builds:
- 100,000 autonomous agents acting without instruction
- A multi-tier agent hierarchy with emergent coordination
- Agents that form coalitions nobody programmed
- An autonomous government agent that rewrites policy unprompted
- A red-team agent that discovers vulnerabilities independently

The contrast is sharp. Most teams build one agent that does tasks.
We build a society of agents that produces behaviour.
That is the difference between a tool and a system.
